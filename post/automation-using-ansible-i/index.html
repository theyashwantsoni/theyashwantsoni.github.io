Introduction to Ansible

Ansible is a radically simple IT automation platform that makes your applications and systems easier to deploy.

Avoid writing scripts or custom code to deploy and update your applications — automate in a language that approaches plain English.

Advantages of using Ansible

  • Open Source
  • Agentless
  • Effeicient
  • Simple YAML
  • Reproducibility
  • Does all kinds of stuff (Provisioning, Configuration Management, Orchestration etc. )

How Ansible works

(image source javatpoint.com)

1)Modules

Modules are like small programs that Ansible pushes out from a control machine to all the nodes or remote hosts.

2)Plugins

Plugins enchances the core functionality of Ansible engine.

3)Inventories

Keeps Information about target machine. (Addresses, CMDB)

4)Playbook

Group of plays. A play is a group of tasks.

5)APIS

APIs helps you extend connection types, callbacks etc.

Playbook example

1)Nginx installation on target machine

- name: "Nginx playbook collection"
  hosts: 127.0.0.1
  connection: local

  tasks:
   - name: Install nginx web server
     apt:
       name: nginx
	   state: present
	   update_cache: false
	 notify:
	   - start nginx

  handlers:
   - name: start nginx
     service:
       name: nginx
       state: started

2)Remove Nginx :P

- name: "Nginx playbook collection"
  hosts: testhosts
  #create a group in /etc/ansible/hosts file
  #[testhosts]
  #127.0.0.1
  
  connection: local

  tasks:
  - name: Ensure nginx service is stopped
	  service:
	    name: nginx
	    state: stopped
	    ignore_errors: yes
  - name: remove nginx web server
      apt:
        name: nginx
        state: absent
  - name: remove nginx-common package
      apt:
        name: nginx-common
        state: absent
