Getting started with Hugo and Github pages

Hugo
Last updated: May 23, 2020

In this article we will discuss about a static site generators.

What is Hugo?

Hugo is one of the most popular open-source static site generators. We can create and serve static content within seconds. The framework is developed using Golang and it serves the “MarkDown” content.

Installation

These steps show installation using .deb package.

First download the package.

curl -s https://api.github.com/repos/gohugoio/hugo/releases/latest \
 | grep  browser_download_url \
 | grep Linux-64bit.deb \
 | grep -v extended \
 | cut -d '"' -f 4 \
 | wget -i -

Then,

sudo dpkg -i hugo*_Linux-64bit.deb

Steps to create your first static site.

All these steps are in official documentation I am just putting them together(here).

1)Create a New Site

hugo new site quickstart

2)Add a Theme

cd quickstart
git init
git submodule add https://github.com/budparr/gohugo-theme-ananke.git themes/ananke
echo 'theme = "ananke"' >> config.toml

3)Add Some Content

The theme we just installed is a blog post theme so just create a post using,

hugo new posts/my-first-post.md

You can see simple markdown here.

4)Serve your content.

hugo server -D

So hello world is that simple.

Deploy to Github pages

1)Change the config.toml and baseURL

publishDir = "docs"
baseURL = "https://<username>.github.io/<repo-name>/"

2)Build your static pages.

hugo -D

3)Create a Repo and push the code

I guess you already know how to do that.

by default the github project page serves the content from /docs folder.

4)Repo settings for static pages

Go to project settings and From Source, select “master branch /docs folder”.

Now visit to https://.github.io//

You can create nice Markdown at Stackedit
