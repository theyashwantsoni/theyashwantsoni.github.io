Dockerizing an Angular app using Nginx

Angular Docker Nginx
Last updated: Apr 10, 2020

What is Docker?

Docker is a containerization tool used to streamline application development and deployment workflows across various environments.

This tutorial shows how to Dockerize an Angular app via Nginx.

We will be using:

  • docker (v18.09.7)
  • node (v12.14.0)
  • angular-cli (v9.0.7)
  • nginx (:alpine from docker hub)

Lets create an Angular app first

Use following command to create a new app.

ng new angular-docker-demo

Change the output path in your angular.json.

"outputPath": "dist"

Now build your app in production mode.

ng build --prod

Creating Nginx config

Angular app is a single page application so all url should need to re-write on index.html.

server {
 listen 80;

 sendfile on;

 default_type application/octet-stream;

 gzip on;
 gzip_http_version 1.1;
 gzip_disable   "MSIE [1-6]\.";
 gzip_min_length  256;
 gzip_vary     on;
 gzip_proxied   expired no-cache no-store private auth;
 gzip_types    text/plain text/css application/json application/javascript application/x-javascript text/xml application/xml application/xml+rss text/javascript;
 gzip_comp_level  9;

 root /usr/share/nginx/html;

 location / {
  try_files $uri $uri/ /index.html =404;
 }
}

Save above code as nginx.conf in your work directory.

Creating Dockerfile

FROM nginx:alpine
MAINTAINER Yashwant Soni<yashwantsoni009@gmail.com>
COPY ./dist /usr/share/nginx/html
EXPOSE 80
#to change the default nginx configuration
COPY ./nginx.conf /etc/nginx/conf.d/default.conf
CMD [ "nginx", "-g", "daemon off;" ]

Save above code as Dockerfile in your work directory.

Building the Docker image

docker build -t angular-container:1.0.0 .

–

docker run -p 3000:80 angular-container:1.0.0 -d

here,

-d option used to run container in detach.

-p (external port):(internal port)

Now go to localhost:3000

To stop the container use following commands,

yashwant@optimus:~/Angular-Docker-Demo$ docker ps
CONTAINER ID    IMAGE           COMMAND         CREATED       STATUS       PORTS         NAMES

–

docker stop <container id>
