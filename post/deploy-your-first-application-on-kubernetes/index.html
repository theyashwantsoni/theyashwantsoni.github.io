Deploy Your First Application on Kubernetes

Kubernetes
Last updated: Mar 27, 2020

What is Kubernetes?

Kubernetes is an open source container orchestration engine for automating deployment, scaling, and management of containerized applications.

This tutorial shows how to Dockerize and deploy a Nodejs app, built with ExpressJs, as Kubernetes cluster.

We will be using:

  • minikube (v1.8.2) - We use Minikube to run a single-node Kubernetes cluster inside a Virtual Machine (VM) on local system. Installation
  • kubectl (v1.17.4) - Kubectl is a command line tool for controlling Kubernetes clusters. Installation
  • docker (v18.09.7) - Docker is a containerization tool used to streamline application development and deployment workflows across various environments. Installation
  • node (v12.14.0) - I guess you are familiar with Node. Installation

Super Simple Node App

We will use ExpressJs server to make a hello-world app.

const express = require('express');
const app = express();
const port = 3000;


app.get('/', (req, res) => res.send('Super simple node app @ codeporn'));


app.listen(port, () => console.log(`Express app listening on port ${port}!`));

Save above code as server.js in your work directory.

Building a Docker Image

To run NodeJS application in Kubernetes, it will need to be shipped as a container.

The following is an example of a Dockerfile for creating a basic container.

FROM node:alpine
MAINTAINER Yashwant Soni<yashwantsoni009@gmail.com>


ADD ./ /opt/webapp/
WORKDIR /opt/webapp
COPY . .
RUN npm i express
EXPOSE 3000
CMD [ "node", "server.js" ]

Save above code as Dockerfile in your work directory.

** You must need root permission to perform following commands in you terminal.

Build docker image using following command.

docker build -t simple-express-app .

Pushing Your Image to a Docker Repository

In order to run the containerized application in Kubernetes it must be available in a Docker repository.

The repository must be accessible to the Kubernetes cluster, as it will be pulled down every time a pod of the image is deployed.

docker login
docker image tag simple-express-app:1.0.0 theyashwantsoni/simple-express-app:1.0.0
docker push theyashwantsoni/simple-express-app:0.0.1

Deploying to Kubernetes

To run kubectl on local:

minikube start

1)Creating a pod

A pod is the smallest unit of a Kubernetes cluster. You can generally replace the word pod with container and accurately understand the concept.

touch simple-express-app-pod.yml

Copy following code and paste it in simple-express-app-pod.yml

apiVersion: v1
kind: Pod
metadata:
 name: simple-express-app
 labels:
  name: simple-express-app
spec:
 containers:
 - name: simple-express-app
  image: theyashwantsoni/simple-express-app:1.0.0
  ports:
  - containerPort: 3000

–

kubectl apply -f simple-express-app-pod.yml
kubectl describe pod simple-express-app
kubectl get pods

To see the minikube dashbaoard:

minikube dashboard

2)Creating a Deployment for Scaling

For services that need to scale, we use Deployment resources.

touch simple-express-app-deployment.yml

Copy following code and paste it in simple-express-app-deployment.yml

apiVersion: apps/v1
kind: Deployment
metadata:
 name: simple-express-app
 labels:
  app: simple-express-app
spec:
 replicas: 5
 selector:
  matchLabels:
   app: simple-express-app
 template:
  metadata:
   labels:
    app: simple-express-app
  spec:
   containers:
   - name: simple-express-app
    image: theyashwantsoni/simple-express-app:1.0.0
    ports:
    - containerPort: 3000

–

kubectl apply -f simple-express-app-deployment.yml
kubectl get deployments
